# Дорожная карта SillyTavern Server для iOS

Этот документ фиксирует запланированные функции после подтверждения запуска
Node 22 и настоящего SillyTavern на физическом iPhone.

## Порядок этапов

1. Завершить экспериментальную сборку NodeMobile 22.
2. Проверить на iPhone запуск Node, HTTP и WebAssembly.
3. Подключить payload SillyTavern и проверить полный запуск.
4. Добавить безопасный импорт и перенос пользовательских данных.
5. Добавить файловый менеджер SillyTavern.
6. Добавить Live Activities сервера и генерации ответа.

## Импорт данных SillyTavern

- выбор ZIP-архива или папки через системный импорт iOS;
- распознавание корня полного архива SillyTavern и отдельного каталога `data`;
- импорт `data/default-user`, `_storage` и `_css`;
- перенос совместимых пользовательских файлов из старых версий;
- исключение Android/Termux `node_modules`, `_cache`, `_webpack`, временных логов
  и старого `cookie-secret`;
- предварительный просмотр изменений;
- резервная копия текущих данных;
- импорт через staging-каталог с атомарной заменой;
- автоматический откат при ошибке.

## Файловый менеджер SillyTavern

### Возможности

- отдельная вкладка «Файлы»;
- дерево каталогов установленного SillyTavern и пользовательских данных;
- поиск по имени, пути и типу файла;
- просмотр текста, JSON, YAML, JavaScript, CSS, HTML и логов;
- просмотр изображений персонажей, аватаров и фонов;
- встроенный текстовый редактор с номерами строк, поиском и отменой изменений;
- проверка синтаксиса JSON перед сохранением;
- создание файлов и папок;
- переименование, перемещение, копирование и дублирование;
- импорт отдельных файлов, папок и ZIP-архивов;
- экспорт файла, папки или пользовательских данных в ZIP;
- отображение размера, даты изменения и свободного места;
- открытие доступной папки через системное приложение «Файлы».

### Удаление и восстановление

- удаление только после подтверждения;
- по умолчанию файлы перемещаются во внутреннюю корзину, а не стираются сразу;
- восстановление из корзины;
- автоматическая резервная копия перед массовым удалением или заменой;
- отдельное подтверждение для необратимой очистки корзины.

### Защита

- обычный режим открывается в пользовательском каталоге `data`;
- runtime, встроенный код приложения и `node_modules` доступны только для чтения;
- изменение служебных файлов возможно только после включения
  «Режима разработчика» и дополнительного предупреждения;
- запрет выхода за разрешённый корневой каталог через `..` и символические ссылки;
- остановка сервера перед изменениями, которые нельзя безопасно выполнять на ходу;
- секреты и API-ключи скрываются в предпросмотре и диагностическом экспорте;
- журнал файловых операций без записи содержимого секретных файлов.

## Live Activities

В настройках обе активности включаются и настраиваются независимо.

### 1. Статус сервера

Состояния:

- запускается;
- работает;
- остановлен;
- приостановлен iOS;
- данные устарели;
- ошибка.

Отображаемые по выбору пользователя данные:

- адрес и порт;
- версия Node;
- версия SillyTavern;
- время работы;
- краткое описание ошибки.

Из активности можно открыть экран сервера. Действия остановки и перезапуска
добавляются только после проверки их надёжной работы через App Intents.

### 2. Генерация ответа SillyTavern

Состояния:

- ожидает ответа API;
- думает;
- генерирует;
- готово;
- отменено;
- ошибка.

Настраиваемые данные:

- имя и аватар персонажа;
- модель;
- прошедшее время;
- количество полученных токенов, если оно доступно;
- уведомление о завершении;
- время показа завершённого результата;
- приватный режим без имени персонажа и модели.

События передаются по цепочке:

```text
SillyTavern -> Node -> Swift -> ActivityKit
```

Активность генерации получает более высокий приоритет, чем постоянный статус
сервера. Если iOS приостанавливает приложение и локальный Node, статус помечается
устаревшим и не должен продолжать показывать ложное «Работает».

## Критерии готовности

### Файловый менеджер

- нельзя выйти за каталоги приложения;
- отмена редактора не меняет файл;
- сохранение выполняется через временный файл и атомарную замену;
- некорректный JSON не перезаписывает исходный файл без отдельного подтверждения;
- удалённый файл восстанавливается из корзины;
- массовая операция имеет проверенную резервную копию;
- импорт архива защищён от Zip Slip и чрезмерного размера распаковки.

### Live Activities

- обе активности можно включать и выключать независимо;
- после повторного запуска приложение находит и завершает устаревшие активности;
- генерация корректно обрабатывает начало, завершение, отмену и ошибку;
- при suspension приложения статус сервера не вводит пользователя в заблуждение;
- приватные данные не появляются при включённом приватном режиме.
